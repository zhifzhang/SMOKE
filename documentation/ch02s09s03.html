2.9.3. Select pollutants from the input files for further SMOKE processing

In version 1.5 and higher of SMOKE, users can specify the valid data (pollutants and activities) using the inventory table discussed in Section 8.10.4, “INVTABLE: Inventory table”, as described in Section 2.9.2, “Assign pollutant names to data input by code numbers”. Smkinven will read only those entries that have a “Y” (for “Yes”) in the Keep column of the inventory table. If a pollutant has an “N” (for “No”) in the Keep column, it will not be output to the SMOKE intermediate files, but will be included in the inventory reports that Smkinven creates. If the pollutant is not listed at all in the inventory table, it will be dropped as well; Smkinven will also write a warning message and will not include the pollutant in any reporting. It is a good work practice for users to put all pollutants in the input inventory in the inventory table even if it the pollutant will not be used for the AQM. Use of the INVTABLE file replaces the use of the SIPOLS and ACTVNAMS files from previous versions of SMOKE.
